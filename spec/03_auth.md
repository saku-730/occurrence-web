# 03. 認証・セッション・CSRF要件

## 認証方式

- ローカル認証を採用する
- メールアドレス + パスワード方式とする
- サーバー側セッション方式を採用する
- OAuth / GitHubログインは MVP 対象外

---

## パスワード要件

- 最小文字数: 8文字
- 最大文字数: 128文字
- 複雑性ルールは設けない
- ハッシュ方式は Argon2id とする
- 平文パスワードを保存してはならない
- 平文パスワードをログに出してはならない

---

## セッション要件

- セッションIDは Cookie に保存する
- Cookie名は `session`
- セッション有効期限は 7日
- rolling session とする
- アクセスごとに有効期限を延長する
- ログアウト時にセッションを無効化する
- パスワードリセット完了時、既存セッションをすべて失効する
- アカウント論理削除時、既存セッションをすべて失効する

---

## Cookie属性

- `HttpOnly` を付与する
- `SameSite=Lax` とする
- 開発環境では `Secure=false` を許可する
- 本番環境では `Secure=true` を必須とする

---

## ログイン

### 正常系

- email は trim + lowercase して照合する
- パスワードが正しければ新しいセッションを作成する
- セッションCookieを返す
- 成功レスポンスは JSON とする

### 異常系

- 存在しない email とパスワード間違いは同じエラーメッセージを返す
- 論理削除済みユーザーはログイン不可
- ログイン失敗回数によるロックは MVP 対象外
- ログイン失敗も監査ログに記録する

---

## CSRF

### 方針

Cookieセッションを使うため、CSRF対策は必須とする。  
MVPでは token 方式を採用する。

### 対象メソッド

以下のHTTPメソッドでは CSRF token を必須とする。

- POST
- PUT
- PATCH
- DELETE

GET / HEAD / OPTIONS は状態変更を行ってはならない。

### 実装方針

- CSRF token を取得するAPIを用意する
- 状態変更APIでは `X-CSRF-Token` ヘッダーで token を送る
- token がない場合は 403
- token が不正な場合は 403

---

## 仮登録

### 正常系

- email を入力して仮登録を開始する
- email は trim + lowercase する
- 未登録の email の場合、仮登録トークンを作成する
- 仮登録トークンの有効期限は 1時間
- トークンはDBに平文保存しない
- 確認メールを送信する

### 同じemailで再送された場合

- 古い未完了トークンは無効化する
- 新しいトークンを発行する

### 登録済みemailの場合

- 画面では「すでに登録済みです」と表示する
- メールは送らない
- 新しい仮登録トークンも作らない

### メール送信失敗

- 仮登録全体を失敗扱いにする
- DB変更はロールバックする
- HTTP 502 を返す

---

## 本登録

### 正常系

- 確認メール中のリンクから本登録画面に遷移する
- 本登録時に username と password を入力する
- 有効な仮登録トークンを検証する
- users テーブルにユーザーを作成する
- 初期ロールは editor とする
- 本登録完了後、自動ログインしない
- ログイン画面に誘導する

### 異常系

- トークンが存在しない場合はエラー
- トークンが期限切れの場合はエラー
- トークンが使用済みの場合はエラー
- 同じトークンで二重登録できない

---

## パスワードリセット

### 申請

- email を入力する
- 存在しない email でも成功風レスポンスを返す
- 存在する email の場合、リセットトークンを作成してメール送信する
- リセットトークンの有効期限は 1時間
- トークンはDBに平文保存しない

### 完了

- 有効なトークンを検証する
- 新しいパスワードを Argon2id でハッシュ化して保存する
- 既存セッションをすべて失効する
- 使用済みトークンは再利用不可

### メール送信失敗

- パスワードリセット申請全体を失敗扱いにする
- DB変更はロールバックする
- HTTP 502 を返す

---

## ユーザー情報変更

MVPで許可するもの。

- editor は自分の username を変更できる

MVP対象外。

- email変更
- email変更確認メール
